## 单用格式

| 写法 | 效果 |
|---|---|
| `!!防剧透!!` | !!防剧透!! |
| `==彩虹渐变==` | ==彩虹渐变== |
| `++下划线++` | ++下划线++ |
| `{注音}(pinyin)` | {注音}(pinyin) |
| `{一|个|字}(yi|ge|zi)` | {一|个|字}(yi|ge|zi) |

标准 Markdown 照常：**加粗** 、*斜体* 、~~删除线~~ 、`代码`。

## 二合一组合

标准语法在外、自定义在内（均可用）：

| 写法 | 效果 |
|---|---|
| `*++斜体下划线++*` | *++斜体下划线++* |
| `*==斜体彩虹==*` | *==斜体彩虹==* |
| `*!!斜体剧透!!*` | *!!斜体剧透!!* |
| `**++加粗下划线++**` | **++加粗下划线++** |
| `**==加粗彩虹==**` | **==加粗彩虹==** |
| `**!!加粗剧透!!**` | **!!加粗剧透!!** |

自定义语法两两组合（均可用）：

| 写法 | 效果 |
|---|---|
| `!!==剧透包彩虹==!!` | !!==剧透包彩虹==!! |
| `!!++剧透包下划线++!!` | !!++剧透包下划线++!! |
| `==++彩虹包下划线++==` | ==++彩虹包下划线++== |
| `++==下划线包彩虹==++` | ++==下划线包彩虹==++ |
| `++!!下划线包剧透!!++` | ++!!下划线包剧透!!++ |

## 重要限制

**标准 Markdown（`**` / `*`）不能放在自定义语法内侧**，因为 markdown 解析器会优先把 `**text**` 转成 `<strong>` 节点，切碎文本后自定义正则无法跨节点匹配。

| 写法 | 文章 | 评论 | 原因 |
|---|---|---|---|
| `**==彩虹==**` | 正常 | 正常 | 标准语法在外层 |
| `==**彩虹**==` | 失效 | 正常 | `**` 在内侧被优先切碎 |
| `!!**剧透**!!` | 失效 | 正常 | 同上 |
| `{*斜体*}(py)` | 失效 | 正常 | `{}` 内的 `*` 被优先解析 |
| `{**加粗**}(py)` | 失效 | 正常 | 同上 |

**安全原则**：自定义语法是**对文本节点的正则替换**，它们不识别 markdown 节点。需要用 `**` `*` 时，把它们放最外层。
