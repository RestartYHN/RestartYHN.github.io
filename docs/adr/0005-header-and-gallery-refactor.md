# ADR 0005：顶栏重构与画廊数据模型收敛（多条目合集）

- 状态：已采纳并验收（2026-06-23）
- 范围：顶栏布局/滚动/导航重构、画廊画师单一事实来源、画廊内容去 i18n、年份筛选、分页控件位置固定

  - `src/data/gallery.template.json`、`src/utils/gallery-utils.ts`
  - `src/pages/[...locale]/appreciation/gallery/[author].astro`、`tags.astro`

## 0005-1：顶栏统一为单条 sticky 常驻 bar

### 背景

原顶栏三组定位方式不一致：logo/工具在 `#main-header`（桌面 `md:relative`，随页滚走），导航是独立**绝对定位浮层**（`left-[46%]`、`z-100`），靠 JS 在滚动时单独加 `fixed`。结果：平板竖屏（≥768 被判为桌面）三组横向挤压重叠；导航 `z-100` > logo/工具 `z-30`，重叠处点击被导航截走；桌面下滑后"只剩导航常驻"。对照 Mizuki 上游，其胶囊本应**包住全部顶栏内容**，Momo 这边把它拆散了——是一处走样。

### 决策

- `#main-header` 改为 **`sticky top-0 z-[100]` 的单条 bar**，全断点一致；删除桌面 `md:relative`、独立 `#nav-bar` 绝对浮层、滚动加 `fixed` 的 JS。
- 移除外层等高 `<div>` 包裹（`MainPageLayout`），使 sticky 能真正吸顶。
- 背景去 `--bg-color` 主题色底，改**纯毛玻璃半透明**适应背景。
- z 一致由结构天然保证，重叠点击问题随之消失。

**教训：与上游同源的组件出问题时，先回上游看"原设计"，往往是 fork 改坏而非需要新设计。多套定位/层级混用是挤压与点击穿透的根因——能统一成一条流式/sticky bar 就别开绝对浮层。**

## 0005-2：常驻取代"下滑隐藏"，移动端对齐桌面

### 背景

移动端原本 `fixed` + 滚动下滑隐藏/上滑唤回；桌面是另一套（导航浮层 fixed）。两端两套滚动行为，割裂。

### 决策

- 两端统一 **常驻不隐藏**：删 `initHeaderScroll` 自动隐藏逻辑与 `.header-hidden` 类。
- `sticky` 自动占位、不盖内容，省掉 `fixed` 的顶部留白计算。

**教训："移动端腾屏"的自动隐藏是默认习惯，但用户更想要"常驻可达"。先确认诉求再决定要不要保留祖传交互。**

## 0005-3：导航图标化 + 断点抬到 lg

### 背景

桌面导航 9 项纯文字，窄屏时 flex 压缩导致**中文逐字竖排**、空隙不均。Mizuki 靠"图标 + 分组下拉 + 配置引擎"省空间，但整套移植成本高（Swup/Tailwind v4/banner 三重耦合）。

### 决策

- **只取"图标"这一招**：导航文字换 fa6 图标（复用移动抽屉已有的同套图标，零新增资源）+ `title`/`aria-label` 悬停提示。图标不换行，竖排问题消失，也更省横向空间——**无需分组下拉**。
- 导航/汉堡断点 **`md`→`lg`**（<1024 走抽屉），"桌面就是桌面"一刀切，平板竖屏不再被当桌面挤压。

**教训：借鉴上游别整套搬，抽出真正解决问题的那一招（图标）用本地现成料实现即可。**

## 0005-4：画廊画师=单一事实来源（author 字段），删冗余 `画师:` 标签

### 背景

每个作品已有权威的 `author` 字段（`gallery-utils.ts` 靠它关联画师、画师注册名 `authors[].name` 双语）。但作品又手写了 `画师:抹茶` 标签：与 `author` 重复、写死中文不随 i18n、在画师页（页面本身就是该画师）重复显示。

### 决策

- **从数据删除所有 `画师:xxx` 标签**，画师一律走 `author` 字段 + `authorInfo.name` 读取；`tags.astro` 的可点击画师名行保留。
- `newgallery.js` 同步去掉"自动追加画师标签"。

**教训：同一信息只存一处。结构化字段（author）优于手写标签——后者既重复又无法随 i18n/注册信息联动。**

## 0005-5：画廊内容降为中文 only

### 背景

i18n 分三层：路由（站点级）、UI 文案 `t()`（站点级）、双语内容 `[normalizedLang]`（局部）。画廊作品标题双写中英、维护麻烦，且英文价值低。

### 决策

- **只动内容层**：画廊作品 `title` 仅存 `zh-cn`（删 `en` 死字段），渲染固定读中文，英文交浏览器翻译；`newgallery` 去 `titleEn` 参数。
- **画师名/简介保留注册双语**（这是权威信息，值得 i18n）。
- 路由层与 `t()` UI 文案**不碰**。
- 标签筛选本就直接读中文标签字符串、未走 i18n，无额外负担；日文画师名首字母分组自动落「其他/#」。

**教训：延续 ADR 0003-4 的 i18n 性价比原则——内容大段交浏览器，权威短字段（人名）才值得双语。改 i18n 要先分清"哪一层"，多数时候只需动最局部那层。**

## 0005-6：年份并入标签筛选——从数据合成维度

### 背景

希望按年份筛选作品，但年份是独立 `year` 字段，不在 `tags` 里。

### 决策

- 构建 `allTags` 和每卡 `data-tags` 时，把 `年份:${year}` 合成进去喂给**现有**筛选管线。侧栏自动多出「年份」维度。
- 数据与筛选 JS 均不改动；卡片年份仍用 `<p>` 显示，不重复成 chip。

**教训：给现成的数据驱动管线"喂"一个合成维度，比改管线本身便宜得多。**

## 0005-7：分页控件位置固定——预留满页高度，非悬浮

### 背景

友链外链分页控件在网格下方，末页卡片少→网格变矮→控件上移"乱跑"。评估过侧箭头、`fixed` 视口底悬浮（B1）、在区块内 sticky（B2）、响应式每页数量等方案。

### 决策

- 关键认知：控件 Y = 网格顶 + 网格高度，要不跳**必须让网格高度恒定**；"下面没内容"只防下游被挤，挡不住网格变矮。B2 在区块内钉住仍需预留高度=等价 A。
- 选 **预留满页高度**：JS 量各页高度**取最大**设为 `grid.min-height`（resize 重算），控件任何页都钉死；末页留白可接受。网格加 `content-start` 防末页单卡被拉伸。统一每页 6 个，不做响应式列数（避免枚举设备/客户端重算）。
- 作品瀑布流维持现状：尺寸各异下 `columns` + `object-contain` 不裁切最合适；列高浏览器已自动平衡，单图浏览交给现成 lightbox，无需改造。

**教训：先把"几何约束"想清楚（控件位置由谁决定）再选方案，能避免在等价方案间打转。能用 CSS 默认行为（多列自动平衡、已有 lightbox）解决的，别自造轮子。**
